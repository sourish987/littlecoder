# Security Policy

If you discover a security issue in LittleCoder, please do not post sensitive details publicly in a GitHub issue.

## What to Include

- a short description of the issue
- how it can be reproduced
- the potential impact
- any suggested mitigation

## Important

- never include real tokens or private credentials
- never include personal machine paths unless necessary
- scrub screenshots and logs before sharing

Until a dedicated security contact is set up, please open a minimal issue without secrets and clearly label it as a security concern.

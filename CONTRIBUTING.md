# Contributing to LittleCoder

LittleCoder is a product-first project. We prefer changes that make the Worker simpler, clearer, and more reliable.

## Ground Rules

- Keep the product simple
- Do not turn LittleCoder into a framework
- Preserve Studio-first usability
- Prefer small pull requests over large rewrites
- Keep changes local-first and predictable

## Good Contributions

- bug fixes
- onboarding improvements
- Studio usability improvements
- safety fixes
- documentation improvements
- better demo quality

## Before Opening a Pull Request

Please make sure:
- the project still starts with `npm start`
- setup still works with `npm run setup`
- no secrets are added
- no runtime folders are committed

## Pull Request Style

- explain the problem clearly
- describe the user-facing effect
- keep scope narrow
- avoid unrelated cleanup

Thanks for helping make LittleCoder better.
